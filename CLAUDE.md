## Project Overview

Tauri v2 plugin that manages long-lived background service lifecycle across Android, iOS, and Desktop. Users implement the `BackgroundService<R>` trait; the plugin handles OS-specific keepalive (Android foreground service, iOS BGTaskScheduler), cancellation, and state management via an actor pattern.

## Code Style

- **Async trait:** `BackgroundService<R>` uses `#[async_trait]` for object safety вЂ” `Box<dyn BackgroundService<R>>`.
- **Non-exhaustive enums:** All public enums (`ServiceError`, `PluginEvent`, `ManagerCommand`) are `#[non_exhaustive]`.
- **Error handling:** `ServiceError` variants: `AlreadyRunning`, `NotRunning`, `Init(String)`, `Runtime(String)`, `Platform(String)`.
- **Serialization:** `#[serde(rename_all = "camelCase")]` on all JSON-facing types. Tagged enum for `PluginEvent`: `{ "type": "started" }`, `{ "type": "stopped", "reason": "..." }`, `{ "type": "error", "message": "..." }`.
- **Naming:** Rust modules/files use snake_case. Public types use PascalCase. JS API uses camelCase.

## Architecture

The plugin uses an **actor pattern** with a single-owner `manager_loop` task:

```
init_with_service(factory)        в†ђ Entry point: creates mpsc channel, spawns actor
  в””в”Ђ ServiceManagerHandle         в†ђ Managed state: holds cmd_tx sender
       в””в”Ђ manager_loop(cmd_rx)    в†ђ Actor: receives ManagerCommand, dispatches
            в”њв”Ђ Start              в†’ handle_start(): check AlreadyRunning в†’ create CancellationToken в†’ increment generation в†’ start keepalive в†’ spawn service task (init в†’ run)
            в”њв”Ђ Stop               в†’ handle_stop(): cancel token в†’ stop keepalive
            в”њв”Ђ IsRunning          в†’ check token slot
            в”њв”Ђ SetOnComplete      в†’ store callback (captured at spawn time)
            в””в”Ђ SetMobile          в†’ store mobile keepalive handle
```

Key mechanisms:
- **Generation counter** (`AtomicU64`): Prevents stale task cleanup from corrupting state after a stopв†’start race.
- **CancellationToken** (`tokio_util`): Cooperative shutdown. Shared via `Arc<Mutex<Option<CancellationToken>>>`.
- **Factory pattern**: `ServiceFactory<R> = Box<dyn Fn() -> Box<dyn BackgroundService<R>>>`. Fresh instance per start.
- **Mobile keepalive**: `MobileKeepalive` trait abstracts Android foreground service and iOS BGTaskScheduler. Set via `SetMobile` command during plugin setup.

## Constraints

1. **DO NOT add business logic** to the plugin вЂ” it is a lifecycle manager only.
2. **DO NOT remove `#[non_exhaustive]`** from any public enum вЂ” preserves API evolution.
3. **Lock Mutex briefly**: create token, store it, drop the lock, THEN spawn the task. Never hold the lock across `.await` or `spawn`.
4. **`CancellationToken` must always be in `tokio::select!`** in any `run()` implementation вЂ” cooperative cancellation is the only shutdown path.
5. **`BackgroundService<R>` trait is object-safe** вЂ” do not add generic methods or associated types that break `Box<dyn BackgroundService<R>>`.
6. **DO NOT reference ServiceRunner, runner.rs, or any old architecture** вЂ” these do not exist in the codebase.

## Platform Behavior

| Platform | Keepalive Mechanism | Auto-restart |
|----------|-------------------|-------------|
| Android | Foreground service with persistent notification (`START_STICKY`) | Yes вЂ” detects `SharedPreferences` flag on Activity launch |
| iOS | BGTaskScheduler with expiration handler (safety timeout: 28s default) | No вЂ” iOS kills BG tasks; cancel listener timeout: 4h default |
| Desktop | Plain `tokio::spawn` вЂ” no OS keepalive needed | No |
