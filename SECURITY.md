## Reporting a Vulnerability

**Do not file public issues for security vulnerabilities.**

To report a security vulnerability, please use the
[GitHub Security Advisory private disclosure form](https://github.com/dardourimohamed/tauri-background-service/security/advisories/new).

### What to include

- A description of the vulnerability and its potential impact.
- Steps to reproduce or a proof-of-concept.
- Affected versions, if known.
- Any suggested mitigations.

### Response timeline

We follow a **90-day coordinated disclosure** timeline:

1. **Acknowledgment** — We will acknowledge your report within 3 business days.
2. **Triage** — We will triage the report and confirm the vulnerability within 7 business days.
3. **Fix** — We aim to release a fix within 30 days of confirmation. Complex issues may take longer; we will keep you informed of progress.
4. **Disclosure** — If 90 days have elapsed since your report and no fix has been published, you may disclose the vulnerability publicly.

### Attribution

Security researchers who report confirmed vulnerabilities will be credited in [CHANGELOG.md](CHANGELOG.md) unless they request otherwise.

### Preferred language

All reports should be submitted in **English**.
